# [Planning] Warvote PRD: 관계형 CRM 및 전략적 인텔리전스 (Phase 1 Focus)

## 1. 개요 및 목적
본 문서는 Warvote 플랫폼의 핵심 기능을 구체화하기 위한 제품 요구사항 명세서입니다. 초기 개발은 지지층 기반의 데이터 수집(CRM)과 여론의 흐름을 파악하는 정보망(인텔리전스) 구축에 집중합니다.

## 2. 모듈 2: 관계형 CRM (Relational CRM) 기능 명세

### 2.1 분산형 데이터 업데이트 시스템
*   **기능:** 관리자가 하부 조직원(A)에게 개별 아이디를 부여하고, 조직원이 직접 자신의 연고자 정보를 등록/관리.
*   **요구사항:**
    *   **입력 방식:** 모바일 웹을 통한 개별 입력 및 엑셀(CSV) 일괄 업로드 지원.
    *   **프로필 항목:** 성함, 연락처, 연령(대), 거주지(동 단위), 성별, 관심사, 직업, 지지 성향(우호/중도/비우호).
    *   **연락처 연동:** 스마트폰 주소록에서 직접 선택하여 업로드하는 기능 (PWA 기술 활용 검토).

### 2.2 지능형 메시지 발송 및 CTA 추적
*   **기능:** 타겟팅된 메시지 발송과 수신자의 반응(클릭)을 실시간으로 추적하여 데이터 정교화.
*   **요구사항:**
    *   **메시지 템플릿:** 텍스트 + 유튜브 링크 + 블로그 링크 조합 지원.
    *   **Personalized Short URL:** 발송 대상 및 추천인(A) 정보를 포함한 고유 단축 URL 자동 생성.
    *   **실시간 분석:** 링크 클릭 시점, 위치(IP 기반 대략적 위치), 클릭 기기 정보 수집.
    *   **데이터 필터링:** 미클릭자 자동 분류 및 다음 발송 제외 대상 리스트 생성.
    *   **Call to Action 지원:** 메시지 하단에 '지인 5명에게 전달하기' 버튼 및 추적 가능한 재배포 링크 제공.

## 3. 모듈 1: 전략적 인텔리전스 (Strategic Intelligence) 기능 명세

### 3.1 멀티 플랫폼 버즈 모니터링
*   **기능:** 유튜브, 틱톡, 인스타그램 및 주요 커뮤니티(맘카페, 정치 커뮤니티)의 실시간 모니터링.
*   **요구사항:**
    *   **영상 플랫폼 감지:** 주요 키워드 기반 유튜브/틱톡 신규 영상 및 부정확한 정보 영상 자동 포착.
    *   **커뮤니티 성향 필터링:** 커뮤니티별(진보/보수/중도) 게시물 노출 빈도 및 반응 온도 차이 분석.
    *   **언론 연계 추적:** 주요 포털(네이버/다음) 기사 배치 시점 및 언론사별 배포 패턴(새벽 게시 등) 대시보드화.

### 3.2 이슈 이동 경로 시각화
*   **기능:** 특정 이슈가 어느 커뮤니티에서 시작되어 어떤 경로로 확산되었는지 시각적 흐름도 제공.
*   **요구사항:**
    *   **이슈 맵:** 최초 발생지 -> 확산 커뮤니티 -> 기사화 단계별 타임라인 시각화.
    *   **댓글 감성 분석:** 원 포스팅과 댓글 간의 감성 일치도(긍부정) 분석을 통한 여론의 실체 파악.

## 4. 기술 요구사항 (Technical Requirements)

### 4.1 데이터베이스 및 API
*   **DB:** 대규모 연락처 및 로그 데이터를 관리하기 위한 관계형 DB(PostgreSQL) + NoSQL(Redis) 하이브리드 구성.
*   **API 연합:** 메시지 발송 API 연동 및 소셜 미디어 API/크롤러 모듈 구축.

### 4.2 보안 및 권한 관리
*   **개인정보:** 연락처 데이터 암호화 저장 및 접근 권한(관리자/조직원) 엄격 분리.
*   **로그 관리:** 데이터 수정 및 메시지 발송 이력의 위변조 방지.

## 5. 단계별 마일스톤
1.  **M1:** CRM 개인별 데이터 입력 및 관리자 통합 뷰 구현.
2.  **M2:** Personalized Short URL 생성 및 클릭 추적 엔진 개발.
3.  **M3:** 주요 커뮤니티 및 유튜브 모니터링 대시보드 프로토타입.

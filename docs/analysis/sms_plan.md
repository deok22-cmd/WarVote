# [기획] 모듈 2: 관계 기반 정밀 타겟팅 CRM (문자/메시지 관리) 플랜

## 1. 개요
*   **목적:** 무분별한 대량 발송으로 인한 스팸 처리, 낮은 도달율 및 비용 낭비 문제 해결. 관계 기반의 정밀 타겟팅을 통해 메시지의 도달률과 투표/참여 전환율을 극대화하는 CRM 시스템 구축.

## 2. 핵심 달성 목표 및 전략적 강화 방안

### 데이터 정교화 (Data Refinement)
*   **수집 항목:** 연령, 거주지, 성별, 관심사, 직업 등 다각적 프로파일 구축.
*   **기획 포인트:** 단순 연락처 수집을 넘어, 각 유권자의 특성과 성향을 태깅(Tagging)하여 맞춤형 메시지 전송 기반 마련.

### 분산 업데이트 시스템 (Distributed Management)
*   **방식:** 중앙 집중식이 아닌, 추천인(A)이 직접 자신의 연고자(가족, 지인 등) 정보를 시스템에 업데이트하고 지속적으로 관리.
*   **기획 포인트:** 다단계 트리 구조 형태의 지지 조직도 구현. 각 지지자가 자신의 관리 그룹을 시각적으로 확인하고 직접 독려할 수 있는 대시보드 제공 필요.

### CTA(Call to Action) 추적 시스템
*   **핵심 기술:** 고유 라벨(Short URL) 생성 및 배포.
*   **성과 측정:** 
    *   링크 클릭률(CTR) 실시간 체크
    *   접속 위치 정보 확인 (법적 허용 범위 내)
    *   미반응자 자동 필터링 및 리타겟팅/제외 라우팅
*   **기획 포인트:** 메시지에 포함된 URL 클릭 여부를 통해 유권자의 관심도를 A/B/C 등급 등으로 자동 분류하는 로직 기획.



## 3. 향후 개발 기획 및 기술 스택 (예정)

### 프론트엔드 (Frontend) - 지지자 및 추천인(A)용 대시보드
*   **기술 스택:** React (TypeScript), Tailwind CSS 등
*   **주요 화면:** 
    *   나의 연고자 리스트 및 정보 수정 페이지
    *   맞춤형 SMS/카카오톡 링크 생성 및 공유(공유하기 버튼) 페이지
    *   내가 보낸 링크의 클릭 통계(CTA 반응률) 대시보드

### 백엔드 (Backend)
*   **기술 스택:** Node.js (TypeScript) 등
*   **주요 API:**
    *   유권자 프로필 데이터 CRUD API
    *   고유 Short URL 생성 및 리다이렉션 로직 적용 API
    *   Short URL 클릭 시 로깅(로그인/클릭/ IP 등) 및 통계 집계 배치 처리를 위한 시스템

### 데이터베이스 (Database)
*   **구조:** 
    *   사용자(User) / 유권자(Voter) 테이블 간의 다대다(N:M) 추천/관계 매핑 테이블 설계.
    *   URL 클릭 로그 데이터 대용량 처리를 위한 로깅 테이블 분리.

## 4. 해결해야 할 과제 (추후 논의)
1.  **개인정보보호법 완벽 준수:** 지인 정보 대리 입력에 따른 개인정보 활용 동의 절차 UI/UX 기획 최적화 방안.
2.  **스팸 필터링 회피:** 통신사 스팸 필터링 정책을 준수하면서 도달률을 높일 수 있는 발송 인프라(혹은 알림톡 연동) 선정.
3.  **성과 보상(Gamification) 요소:** 자발적 확산을 독려하기 위한 합법적 선 안에서의 배지, 랭킹 시스템 등의 기획.

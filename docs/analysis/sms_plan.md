# [기획] 모듈 2: 관계 기반 정밀 타겟팅 CRM (문자/메시지 관리) 플랜

## 1. 개요
*   **목적:** 무분별한 대량 발송으로 인한 스팸 처리, 낮은 도달율 및 비용 낭비 문제 해결. 관계 기반의 정밀 타겟팅을 통해 메시지의 도달률과 투표/참여 전환율을 극대화하는 CRM 시스템 구축.

## 2. 사용자 권한 및 용어 정의 (Role Definitions)
시스템 내 혼선을 막기 위해 사용자 계정 권한과 용어를 다음과 같이 엄격히 분류하여 통일합니다.
1. **전체 관리자 (System Admin):** 이 시스템의 최고 총괄 관리자. 하위 '관리자'의 계정을 생성하고 폐쇄할 수 있는 권한을 가집니다.
2. **관리자 (Camp Admin):** '전체 관리자'가 승인/생성해 준 계정(예: 홍길동캠프, 홍길동 후보). 시스템 내 하위 설정 및 관리를 담당하며, 캠프에 연동된 '연고자'를 대상으로 한 실제 **문자메시지(SMS/알림톡) 발송 권한**을 독점적으로 가집니다.
3. **추천인 (Supporter):** 특정 '관리자(캠프)' 소속으로 활동하는 선거운동원/자원봉사자. 관리자가 신원을 인증하고 계정을 생성해줍니다. 자신의 선거구 내에 있는 본인의 지인 데이터(연고자)를 시스템에 업로드, 수정, 삭제하는 데이터 구축 권한만을 가집니다. 문자 발송 권한은 없습니다.
4. **연고자 (Contact):** 3번의 '추천인'이 자신의 주소록 등을 통해 업로드한 지인 대상자. 시스템 접속 권한이 없는 순수 발송 대상 타겟(데이터)입니다.

## 3. 핵심 달성 목표 및 전략적 강화 방안

### 데이터 정교화 (Data Refinement)
*   **수집 항목:** 연고자명, 연락처, 연령, 성별, 직업 등 다각적 프로파일 구축. (기존 '관심사' 항목 제외)
    *   **거주지 세분화:** 광역시별, 기초단체별, 동별, 상세주소로 체계적으로 나누어 기입.
    *   **지지여부 (스탠스):** 적극지지, 소극지지, 중도, 소극반대, 적극반대의 5단계로 세분화. 이 스탠스 분류 기준에 대한 설정 및 판단 책임은 전적으로 **'관리자'**에게 있습니다. (예: 관리자의 설정에 따라 "발송 메시지 내 링크 클릭률이 70%가 넘는 연고자는 '적극지지'로 분류한다"와 같은 전략적 룰 적용 기능 지원)
*   **기획 포인트:** '관리자'가 메시지를 발송할 때, 지지여부 데이터 옆에 있는 [선택] 체크박스를 활용하여 특정 스탠스의 사람들에게만 발송할 수 있도록 제어하는 화면 구성을 마련.

### 분산 업데이트 시스템 (Distributed Management)
*   **데이터 획득 및 DB 구축 방식:** 캠프 주도의 중앙 집중식이 아닌, **'추천인'**이 직접 자신의 **'연고자'** 데이터를 수집하여 시스템에 업데이트합니다.
    1.  **엑셀 일괄 제출:** '추천인'이 양식에 맞춘 엑셀 데이터를 제출하면, '관리자'가 웹 관리자 페이지에서 이를 일괄 업로드합니다.
    2.  **모바일 주소록 연동 업로드:** '추천인'이 본인의 휴대폰(앱/웹)에서 로그인하여 스마트폰 주소록을 연동한 뒤, 타겟을 선택하여 일괄 업로드합니다. (업로드 후에도 추천인이 연고자의 주소, 나이, 이름 등 수정 가능)
    3.  **개별 직접 입력 (Manual):** '추천인'이 모바일 등에서 로그인 후 연고자 데이터를 하나씩 직접 타이핑하여 등록합니다.
    4.  **필수 기입 사항:** 위 모든 업로드/작성 과정에서 연고자의 **'이름'과 '휴대폰 번호'는 반드시 필수 기입 항목**입니다.
    5.  **본인 리스트 관리:** '추천인'은 언제든 로그인하여 본인이 업로드한 연고자 리스트를 열람하고, 삭제/추가 작업을 자유롭게 수행할 권한이 있습니다.
    6.  **삭제 리스트 관리자 보관 (Soft Delete):** '추천인'이 본인 권한으로 연고자 데이터를 삭제하더라도, '관리자' 페이지에서는 '삭제 리스트 목록'으로 별도 이관되어 상위 레벨의 데이터 관리가 가능해야 합니다.
*   **기획 포인트:** 관리자-추천인-연고자로 이어지는 명확한 권한 및 트리 구조 구현. 추천인에게는 자신이 모집한 연고자 현황을 볼 수 있는 실적 대시보드가 주어집니다.

### CTA(Call to Action) 기반 Short URL 및 성과 추적 시스템
*   **고유 라벨(Short URL) 생성 및 메시지 자동 치환:**
    1.  **다양한 경로 단축 지원:** 대상자에게 발송할 블로그 주소, 유튜브 영상 링크, 기타 웹사이트 주소 등의 원본 링크를 시스템 내에서 입력창에 넣으면 즉시 Short URL로 자동 생성해 줍니다.
    2.  **메시지 내 자동 치환(Macro) 배포:** 발송 문자 메시지 작성창에서 'Short URL 삽입 위치'를 지정해 두고 내용을 작성하면, 실제 메시지가 발송될 때 수신 대상자 1명 1명마다 고유하게 추적되는 개별 URL이 자동으로 생성(치환)되어 발송됩니다.
*   **성과 측정 및 관리 (메시지별 & 개별 발송 이력 추적):** 
    1.  **단축 URL 심층 추적:** 대상 수신자가 각각 발송받은 고유 URL 링크를 클릭했을 때, '누구의 추천인 정보인지', '해당 메시지를 클릭했는지/안 했는지' 여부와 함께 **해당 링크 목적지(블로그 등)에서의 체류 시간**까지 기술적으로 추적 및 수집합니다.
    2.  **메시지 캠페인별 결과창:** 특정 메시지 발송 건을 기준으로 하여 총괄 통계(발송 대상자 수, 전송 성공자 수, 전송 성공률, 반응(클릭) 수, 미반응 수, 최종 반응 비율, 평균 체류시간)를 하나의 창에서 확인합니다.
    3.  **수신자(대상자)별 세부 결과 및 이력 기록:** 위 2번 항목에서 볼 수 있는 통계 및 세부 지표들을 개별 연고자의 '개인 데이터 리스트' 뷰에서도 동일하게 열람할 수 있어야 합니다. 더불어 메인 연락처 DB 리스트 자체에 **수신자의 총 누적 반응(클릭)수**가 기본 데이터로 노출되어야 합니다. 수신자가 과거 어떤 내용은 클릭했고 어떤 내용은 무시했는지를 뷰어에서 교차 조회할 수 있습니다.
    4.  **연속 미반응자 전략적 관리 (미발송 대상 분류):** 누적 이력 데이터를 바탕으로 3~4차례 문자를 발송했음에도 계속 미클릭인 대상자 리스트를 명확히 필터링 해 줍니다. 이들을 향후 '미발송 대상'으로 제외할지 계속 공략할지는 시스템이 자동화하지 않고 캠프 관리자가 데이터를 근거로 '전략적 판단'을 결정합니다. 이후 판단 결과에 따라 결과창 내에서 직관적으로 대상을 지정/체크할 수 있습니다.
    5.  **데이터 도출 (Export):** 이 모든 메시지별, 대상자별 발송 결과 및 통계 데이터는 관리자 또는 추천인이 별도 분석을 하거나 보고할 수 있도록 **엑셀(Excel) 문서 포맷으로 다운로드**할 수 있어야 합니다.
*   **기획 포인트:** '체류시간'이라는 강력한 지표 추가 및 엑셀 출력 지원, 그리고 다수의 메시지가 클릭당 1명씩 매핑되는 고도화된 자동 병합 발송 설계가 해당 시스템의 코어 경쟁력입니다.



## 4. 향후 개발 기획 및 기술 스택 (예정)

### 프론트엔드 (Frontend) - 추천인용 및 관리자용 대시보드
*   **기술 스택:** React (TypeScript), Tailwind CSS 등
*   **주요 화면:** 
    *   나의 연고자 리스트 및 정보 수정 페이지
    *   맞춤형 SMS/카카오톡 링크 생성 및 공유(공유하기 버튼) 페이지
    *   내가 보낸 링크의 클릭 통계(CTA 반응률) 대시보드

### 백엔드 (Backend)
*   **기술 스택:** Node.js (TypeScript) 등
*   **주요 API:**
    *   유권자 프로필 데이터 CRUD API
    *   고유 Short URL 생성 및 리다이렉션 로직 적용 API
    *   Short URL 클릭 시 로깅(로그인/클릭/ IP 등) 및 통계 집계 배치 처리를 위한 시스템

### 데이터베이스 (Database)
*   **구조:** 
    *   사용자(User) / 유권자(Voter) 테이블 간의 다대다(N:M) 추천/관계 매핑 테이블 설계.
    *   URL 클릭 로그 데이터 대용량 처리를 위한 로깅 테이블 분리.

## 5. 비즈니스 상용화 과금 모델 (B2B Billing System)
향후 상용화 단계에서 '관리자(캠프)'를 대상으로 한 수익 및 과금 체계는 다음과 같습니다.

1.  **기본 서비스 사용료 (초기 세팅 및 계정 가입비):**
    *   **계정 가입비:** 선거 캠프(관리자) 1계정당 부여되는 시스템 초기 세팅 비용입니다. 
    *   **데이터 볼륨 과금 (Tier):** 계정당 저장/관리하는 데이터 수량에 따라 기본 요금이 결정됩니다. (예: 예상 데이터 5만 개 이하 기본 요금, 초과 시 초과 볼륨에 비례하여 추가 가입비 지불)
    *   **포함 내역:** 앞서 기획된 모든 관리 기능(데이터 관리, 수동 입력 지원, Short URL 기반 발송 추적 등)에 대한 이용료는 이 가입비에 포함되어 제공됩니다.
2.  **실 발송 비용 (선불 충전 방식):**
    *   문자메시지(또는 알림톡) 실 발송 요금은 플랫폼 이용료와 분리됩니다. 관리자가 사전에 금액을 미리 결제하고 충전해 두면, 실제 발송을 진행할 때마다 충전금액 한도 내에서 건당 차감되는 선불(Pre-paid) 구조를 따릅니다.

## 6. 해결해야 할 과제 (추후 논의)
1.  **개인정보보호법 완벽 준수:** 지인 정보 대리 입력에 따른 개인정보 활용 동의 절차 UI/UX 기획 최적화 방안.
2.  **스팸 필터링 회피:** 통신사 스팸 필터링 정책을 준수하면서 도달률을 높일 수 있는 발송 인프라(혹은 알림톡 연동) 선정.
3.  **성과 보상(Gamification) 요소:** 자발적 확산을 독려하기 위한 합법적 선 안에서의 배지, 랭킹 시스템 등의 기획.

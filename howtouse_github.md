# Warvote 협업을 위한 GitHub 사용 가이드

이 문서는 **Warvote** 프로젝트 저장소를 공유하고, 팀원들이 효율적으로 협업하기 위한 가이드라인을 제공합니다.

## 1. 초기 세팅 (가져오기)

저장소를 처음 내 컴퓨터로 가져오려면 아래 명령어를 사용합니다.

```bash
git clone https://github.com/deok22-cmd/WarVote.git
cd WarVote
```

## 3. Antigravity(AI)를 활용한 자동화 가이드 (권장)

직접 명령어를 입력하는 대신, AI 비서인 **Antigravity**에게 다음과 같이 요청하여 모든 Git 업무를 대신 수행하게 할 수 있습니다.

### 3.1 주요 요청 예시
*   **최신 코드 동기화:** `"GitHub에서 최신 코드 pull 해줘."`
*   **작업 내용 업로드:** `"지금까지 코딩한 내용 '회원가입 기능 구현'이라는 메시지로 commit 하고 push 해줘."`
*   **충돌 해결:** `"git pull 하다가 충돌이 났는데, docs/prd.md 내용이랑 비교해서 해결해 줘."`
*   **브랜치 관리:** `"신규 기능을 위한 feature/mapping 브랜치 만들어서 이동해 줘."`

---

## 4. 협업 시 Antigravity 활용 팁

### 4.1 일관성 있는 개발 (BMad 방법론)
Antigravity는 프로젝트의 `docs/` 폴더에 있는 기획/설계 문서를 읽고 코드를 짭니다. 팀원 중 한 명이 문서를 수정했다면, Antigravity에게 다음과 같이 말하세요.
> **"수정된 `docs/prd.md` 내용을 확인하고, 그에 맞춰서 API 코드를 업데이트해 줘."**

### 4.2 코드 리뷰 및 설명 요청
친구가 짠 코드가 이해되지 않을 때 Antigravity에게 물어볼 수 있습니다.
> **"친구가 방금 push 한 코드의 로직을 설명해 주고, 개선할 점이 있는지 비판적으로 검토해 줘."**

---

## 5. 수동 업무 프로세스 (AI를 사용할 수 없는 경우)

(이하 기존 Git 명령어 가이드...)

1.  **작업 시작 전:** `git pull origin main`
2.  **커밋:** `git commit -m "메시지"`
3.  **푸시:** `git push origin main`


본 프로젝트는 **BMad 방법론**에 따라 `docs/` 폴더에 기획 및 설계 문서가 최신화되어 있습니다.
*   새로운 기능을 코딩하기 전, 반드시 `docs/prd.md`와 `docs/architecture.md`를 먼저 읽어보세요.
*   문서가 수정되었다면, AI(Antigravity 등)에게 "수정된 문서를 바탕으로 코드를 짜줘"라고 요청하면 더 정확한 결과물을 얻을 수 있습니다.
